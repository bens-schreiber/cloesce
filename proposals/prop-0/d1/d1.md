Two scenarios can occur in a D1 to D1 relationship:

1. Both models are in the same database (e.g., `ModelA` and `ModelB` are both in `DbA`)
2. The models are in different databases (e.g., `ModelA` is in `DbA` and `ModelB` is in `DbB`)

In either case, the relationship can be `one` or `many`.

In this proposal, we will no longer `JOIN` Models together in SQL. Instead, we will query the base model first, and then query the related models in a separate query, regardless of whether they are in the same database or not. This cuts down on cartesian explosion, and allows cross-database relationships and same-database relationships to be handled in the same way.

# D1 Notes

Even though the primary key parameters are always supplied in the `get` operation, we cannot immediately resolve the relationships in a single query even if they depend only on the primary key of the model being queried. This is because we must first guarantee that the base model exists before we can query for its relationships. If the base model does not exist, the runtime should halt the query and return an error.
