# Runtime + E2E + Examples — New-Syntax Completion Plan

Handoff for finishing the Cloesce **new-syntax** migration across the three
surfaces the compiler pass deliberately left out of scope: the **TypeScript
runtime** (`src/runtime/ts`), the **e2e test fixtures + tests** (`tests/e2e`),
and the **examples** (`examples/reddit`, `examples/weather`). Plus: **add a new
e2e test that exercises `[header]` params end to end.**

The compiler (frontend/semantic/idl/codegen/orm/migrations) is fully migrated and
**green**. This work makes the generated code actually *run* under the new syntax,
and lands the runtime half of `[header]` support (extraction from the request),
which codegen already emits the client/backend *shape* for.

---

## 0. What the new syntax is (only the parts that reach these surfaces)

- API methods declare params in the **body**, not the signature:
  `get foo -> T { a: int  b: string }`.
- **Instance vs static** is a leading `self` prefix on the method header
  (NOT a body block — the `source { }` block was removed):
  - `get x -> T { }` → **static**
  - `self get x -> T { }` → **instance**, `Default` source
  - `self(Custom) get x -> T { }` → **instance**, source `Custom`
- `[inject X]` is gone; injection is an `inject { ... }` block in the method body.
  Durable context uses `::`: `MyDurable::id(value)`, spider `MyDurable::{a(x), b(y)}`,
  empty `MyDurable::{}`.
- `[header]` tags a param as coming from a **request header** instead of the body:
  `[header] Authorization: string`.
- `vars` → `var`; `optional` → `option`.
- KV/R2/DO templates declare params in the body:
  `entry -> json { id: int  "cache/{id}" }` (no `entry(id: int) -> json`).
- KV/R2/DO *references* on a model keep the `Binding::selector(localFields)` form:
  `kv MyKv::someValue(id1, id2) { fieldName }`; a no-arg template ref is bare:
  `kv GlobalDo::config { fieldName }`.

`src/compiler/compiler-test/src/src_str.rs` (`COMPREHENSIVE_SRC`) is the canonical
example of every construct — consult it whenever unsure of exact shape.

## 1. IDL shape changes the runtime must mirror (the crux)

`src/runtime/ts/src/cidl.ts` is a **hand-maintained** TypeScript mirror of the Rust
IDL (`src/compiler/idl/src/lib.rs`). The compiler pass changed `ApiMethod.parameters`
from `Vec<ValidatedField>` to `Vec<ApiMethodParam>`, where:

```rust
enum ParamSource { Body, Header }          // default Body
struct ApiMethodParam { field: ValidatedField, source: ParamSource }
```

`cidl.ts` still has `parameters: ValidatedField[]`. This is the single most
important runtime change — everything else in the router flows from it.

**Update `cidl.ts`:**
```ts
export type ParamSource = "Body" | "Header";
export interface ApiMethodParam {
  field: ValidatedField;
  source: ParamSource;
}
export interface ApiMethod {
  // ...
  parameters: ApiMethodParam[];   // was ValidatedField[]
  // ...
}
```
Note: `DataSourceMethod.parameters` and `DataSourceGetMethodParam.parameter`
are **unchanged** (still `ValidatedField` / the get-param wrapper) — only
`ApiMethod.parameters` changed. Don't touch the data-source shapes.

Confirm the serialized `cidl.json` fixtures agree: the compiler now emits
`{"field": {...}, "source": "Body"|"Header"}` for each API method param. Every
`tests/e2e/fixtures/*/cidl.json` is regenerated by the build (see §4), so you do
not hand-edit them — but the mirror must match what the compiler writes.

## 2. Runtime router changes (`src/runtime/ts/src/router/router.ts`)

All API-method param access goes through `route.method.parameters`, which is now
`ApiMethodParam[]`. Every `p.name`/`p.cidl_type` on an **API** param becomes
`p.field.name`/`p.field.cidl_type`.

### 2a. Thread the new shape (mechanical)
Find each place that iterates `route.method.parameters` (the CRUD/data-source
param loops using `.get.parameters[i].parameter` are a *different* type and stay):
- `validateParams` (~line 321–372): `requiredParams = route.method.parameters`.
  - `requiredParams.find((p) => ... p.cidl_type === "Stream")` → `p.field.cidl_type`.
  - `params[streamParam.name]` → `params[streamParam.field.name]`.
  - `requiredParams.every((p) => p.name in params)` → `p.field.name`.
  - `validateField(p, params[p.name])` → `validateField(p.field, params[p.field.name])`
    (`validateField` takes a `ValidatedField`, so pass `p.field`).
  - `hydrateType(validatedRaw, p.cidl_type, ...)` → `p.field.cidl_type`.
- Anywhere else `.parameters` on an `ApiMethod` is read (grep the whole
  `src/runtime/ts/src` tree for `.parameters` and check each is an ApiMethod vs a
  DataSourceMethod).

### 2b. Header extraction (the deferred feature — implement it now)
In `validateParams`, params flagged `source === "Header"` must be read from
`request.headers`, NOT from the body/searchParams. Sketch:
- Build the params map as today (searchParams for GET, body for non-GET).
- Then, for each `p` in `requiredParams` where `p.source === "Header"`, pull the
  raw value from `request.headers.get(p.field.name)` and place it into `params`
  under `p.field.name` (so the existing missing-param check and `validateField`
  loop work unchanged). Header values are strings — they flow through the same
  `validateField` → `hydrateType` path (which already coerces string→int/date/etc.
  for GET query params), so date/int/bool header params coerce correctly.
- A missing required header should produce the same 400 as a missing body param
  (the existing `requiredParams.every(...)` check will catch it once headers are
  merged in; verify the message is sensible).
- Header params must be **excluded** from the body/searchParams requirement: since
  they're merged into `params` from headers before the `.every()` check, this is
  automatic — but make sure a `[header]` param on a POST isn't *also* expected in
  the JSON body (it isn't required there because we populate it from the header
  first). Match the client codegen, which sends header params via
  `__$headers` and NOT in `__$payload` / searchParams
  (see `src/compiler/codegen/templates/client.ts.jinja`).

### 2c. Runtime unit tests (`src/runtime/ts/tests/router.test.ts`)
- `tests/builder.ts` / `tests/common/*` likely build `ApiMethod` fixtures in the
  old `parameters: ValidatedField[]` shape — update the builder(s) to wrap params
  as `{ field, source: "Body" }`. Grep the `tests/` tree for `parameters:` and
  `cidl_type` in ApiMethod construction.
- Add a router unit test for a `Header`-sourced param: a method with one
  `[header]`-equivalent param (`source: "Header"`), assert the value is read from
  `request.headers` and a missing header 400s.

Gate: `cd src/runtime/ts && pnpm test` (or the repo's runtime test script) green
before moving on. `pnpm -w run build` / `tsc` on the runtime must pass — check the
runtime's own build script.

## 3. Convert e2e fixtures to the new syntax (`tests/e2e/fixtures/*`)

Every fixture has a `schema.cloesce` (+ generated `backend.ts`, `client.ts`,
`cidl.json`, `dist/workers.js`) and a hand-written `workers.ts` impl. **The
`schema.cloesce` files are still old syntax** (e.g. `services/schema.cloesce` uses
`[inject InjectedThing]` and `get method() -> string`). Convert every
`schema.cloesce` under `tests/e2e/fixtures/`:

Mechanical rewrites (same as the compiler-pass conversions — see
`COMPREHENSIVE_SRC` and the converted `src/compiler/semantic/tests/*` for exact shape):
- `[inject A, B] verb m(self, x: T) -> R` →
  ```
  self verb m -> R {
      x: T
      inject { A  B }
  }
  ```
  (bare `self` for the default source; drop `self` entirely for static methods).
- `[inject Do(a)]` → `inject { Do::shard(a) }` (name the shard field);
  `[inject Do()]` → `inject { Do::{} }`.
- `[source S] self` → `self(S)`.
- KV/R2/DO templates: `name(p: T) -> U { "fmt" }` → `name -> U { p: T  "fmt" }`.
- `vars {` → `var {`; `optional` → `option`.

The **impls in `workers.ts` should not need signature changes** — codegen already
produces the same `Api.Model.method` types (params are still positional args, header
params included). But re-generate and re-typecheck each (see §4); if a generated
signature shifted (e.g. a param that moved body→header), fix the impl to match.

Do NOT hand-edit generated `backend.ts` / `client.ts` / `cidl.json` / `dist/` —
they come from the compiler.

## 4. Regenerate + run e2e

The e2e harness compiles each fixture's `schema.cloesce` with the built `cloesce`
binary, generates `backend.ts`/`client.ts`/`cidl.json`, bundles `dist/workers.js`,
then boots wrangler per fixture and hits it over HTTP.

- Build the compiler first: **`make build-src`** (per repo rule — not raw cargo).
- Regenerate fixtures: find the codegen step the e2e setup uses
  (`tests/e2e/src/globalSetup.ts` / `setup.ts` / fixture `cloesce.jsonc`) and run
  the project's regeneration path so every fixture's generated files refresh from
  the new-syntax schema.
- **e2e workflow rules (important, from prior sessions):**
  - Run tests **one at a time**, not the whole suite at once.
  - **Kill stale wrangler processes** between runs (they hold ports).
  - Typecheck first, but **do not run a full-package `tsc`** — it's slow; scope to
    the fixture, or rely on the wrangler build. Prefer `wrangler build`-level checks
    over a repo-wide `tsc`.
  - Format any hand-written TS with `pnpm -w run fmt` (oxfmt), never prettier.
- The full `make test` includes runtime + e2e; by the end of this work it should
  pass (that was the piece the compiler pass couldn't reach).

## 5. New e2e test: `[header]` params

Add a fixture + test that proves header params work end to end through the
generated client → HTTP headers → runtime extraction → impl.

- **Fixture** `tests/e2e/fixtures/headers/`:
  - `schema.cloesce`: a model with an API method taking at least one `[header]`
    param and one body param, e.g.:
    ```
    model HeaderService {}

    api HeaderService {
        post echo -> string {
            [header]
            X_Tenant: string

            payload: string
        }
    }
    ```
    Consider also a GET with a header param (headers apply regardless of verb) and
    a typed header (e.g. `[header] X_Count: int`) to prove string→int coercion.
  - `workers.ts`: impl returns something derived from BOTH the header and body
    param, so the assertion can prove the header value actually arrived
    (e.g. `` `${X_Tenant}:${payload}` ``).
  - `cloesce.jsonc` / `wrangler.toml`: copy the shape from an existing simple
    fixture (e.g. `services`).
- **Test** `tests/e2e/tests/headers.test.ts` (model on `services.test.ts`):
  - Call via the generated client (which sends the header param as an HTTP header)
    and assert the round-tripped value.
  - Add a raw `fetch` case that omits the required header and asserts a 400, and
    one that sets the header directly, to prove extraction is header-sourced (not
    body-sourced).
  - If a typed header is included, assert coercion (int/date parsed from the string
    header).
- Register the fixture wherever the e2e harness enumerates fixtures for codegen
  (mirror how `services` is wired).

## 7. Gating / order / rules

1. **Runtime first** (`cidl.ts` mirror → router threading → header extraction →
   runtime unit tests green). Nothing downstream is trustworthy until the mirror
   matches the compiler output.
2. **e2e fixtures** convert → regenerate → run **one test at a time**, green.
3. **New header e2e** fixture + test, green.
4. **Examples** convert → build/typecheck.
5. Full `make test` as the final gate (should now pass; if runtime/e2e still fails,
   it's in scope to fix).

Rules to honor:
- Build the compiler with **`make build-src`**, never raw `cargo build`.
- **Never `git stash`** (hard rule; binds any sub-agents too).
- e2e: one test at a time; kill stale wranglers; don't run full-package `tsc`.
- Format TS with `pnpm -w run fmt` (oxfmt), never prettier.
- Don't hand-edit generated files (`backend.ts`, `client.ts`, `cidl.json`, `dist/`);
  regenerate them.
- Delete tests that only asserted removed old-syntax behavior rather than contorting
  them; convert tests that assert still-valid behavior.
- Don't change the compiler crates — they're green and out of scope here. If you
  believe a compiler change is required, stop and flag it rather than editing.
- Don't touch unrelated behavior; this is a syntax migration + the runtime half of
  `[header]`.
